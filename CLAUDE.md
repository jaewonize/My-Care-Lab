# CLAUDE.md — My Care Lab: 나만의 솔루션 만들기

## 프로젝트 개요

My Care Lab의 PoC B.
사용자가 에이전트와 대화형으로 건강 솔루션을 co-creation하고,
생성된 플랜 기반으로 넛징과 보상이 작동하는 루프를 시연한다.
시나리오 예제: 체중감량.

---

## 기술 스택

- 단일 HTML 파일 (순수 HTML/CSS/JS)
- Claude API: `claude-sonnet-4-20250514`, `/v1/messages`
- 모바일 프레임: 360px × 780px (갤럭시 S25 스타일, border-radius 44px)
- Canvas API: 체형 슬라이더 Warp 변형 (per-scanline)
- localStorage: 플랜 저장, 보상 이력 (키: `mycarelab_plan`, `mycarelab_rewards`)

## 화면 구조 & 라우팅

화면 전환은 CSS translateX 슬라이드 애니메이션 (iOS 스타일).
`.screen.active` / `.screen.exit-left` 클래스로 제어.

```
screen-home      홈 — 기존 솔루션 카드 + "나만의 솔루션 만들기" 진입
screen-chat      에이전트 대화 — co-creation 플로우
screen-reward    보상 페이지 — 카드 플립 + 컬렉션 그리드
```

**홈 화면 구성**
- 상단: hero (인사 + 타이틀)
- 더미 솔루션 카드 2개 (기존 플랜 예시)
- 플랜 트래킹 넛징 카드 (`#nudge-mount`) — 플랜 확정 후 표시
- "나만의 솔루션 만들기" create-card (진입점)

## Co-creation 플로우 (screen-chat)

### 단계 구조
`STATE.step`으로 관리. 각 단계 완료 후 다음 단계로 진행.
뒤로가기: 각 단계 상단 topbar의 `back` 버튼 + 플랜 견적 카드 내 `← 이전 단계로` 링크.

```
step 0  인트로 — 에이전트 환영 메시지
step 1  신체 데이터 입력 — 키/체중/체지방률/근육량/나이/성별
step 2  사진 업로드 + Canvas Warp 슬라이더 — 목표 체형 설정
step 3  타겟 날짜 선택 — 메모리 이벤트 옵션 + 프리셋 + 직접입력
step 4  플랜 견적 카드 — 개인화 역산 결과 표시 + 수락/강화/순화
step 5  플랜 확정 — 솔루션 카드 생성 후 홈으로 복귀
```

### 뒤로가기 구현 현황
- topbar `.back` 버튼: 각 step에서 이전 step으로 이동
- 플랜 견적 카드 (step 4): `<button data-step-back>← 이전 단계로</button>` — `canBack` 조건부 렌더링
- `resolve('back')` / `resolve('forward')` Promise 패턴으로 step 흐름 제어

### Step 2 — Canvas Warp 슬라이더

**구현 방식**: per-scanline 수평 압축 (외부 AI 없음, 브라우저 Canvas API만 사용)

```javascript
// 3구간 압축 비율
허리/복부 (yRatio 0.40~0.65): 최대 28% 압축 (peak at 0.525, 가우시안 감쇠)
어깨      (yRatio 0.20~0.35): 최대 6% 압축
허벅지    (yRatio 0.70~0.90): 최대 10% 압축
```

- 사진 미업로드 시 SVG 실루엣 폴백 자동 표시 (`makeSilhouetteDataURL()`)
- 슬라이더 0~100 → `sliderValue` 0.0~1.0 변환
- 슬라이더 > 0.5 시 `.is-goal` 클래스 토글 (목표 도달 시각 피드백)

**readout 계산** (`updateReadout()`):
- 슬라이더 1.0 기준: 체중 최대 -10kg, 체지방률 최대 -10%, 근육량 최대 +0.5kg
- `STATE.goal = { weight, fat, muscle, slider }` 실시간 업데이트

### Step 3 — 타겟 날짜 선택

메모리 이벤트 더미 데이터 (실제 연동 전 시연용):
```
○ My Wedding — 2026년 8월 15일 (D-87)
○ Trip to Hawaii — 2026년 10월 3일 (D-136)
○ 3개월 후
○ 6개월 후
○ 1년 후
○ 직접 입력
```

### Step 4 — 개인화 플랜 견적

**에이전트가 입력 데이터를 분석해서 카테고리별 목표치를 역산한다.**
룰 기반 고정값이 아니라, 이 사람의 현재 상태에서 무엇이 병목인지를 먼저 진단한다.

#### 개인화 역산 로직 (미구현 → 구현 필요)

현재 코드는 카테고리 목표치가 고정값으로 하드코딩되어 있음:
```
잘 태우기: +300kcal / 8,000보 (고정)
잘 자기:   7시간 / 점수 75↑ (고정)
잘 빠지기: 주 -X kg (목표 감량 ÷ 기간으로 역산됨 — OK)
잘 버티기: 스트레스 지수 60 이하 (고정)
```

**개인화로 교체해야 할 로직:**

```
칼로리 적자 역산
- 필요 총 칼로리 적자 = 목표 감량(kg) × 7,700kcal
- 일일 필요 적자 = 총 적자 ÷ 기간(일)
- 현재 활동량 기준 잘 태우기 목표치 산출
  → 이미 많이 움직이는 사람: 추가 소모 목표치 낮게
  → 좌식 위주: 추가 소모 목표치 높게

수면 병목 진단
- 현재 수면 < 6h → 수면 시간 목표 개인 역산 (병목 플래그 ON)
- 수면 시간 충분 + 점수 낮음 → 수면 질 목표 설정 (병목 플래그 ON)
- 수면 양호 → 잘 자기 목표치 낮게 (유지 수준, 병목 플래그 OFF)

스트레스 병목 진단
- 스트레스 지수 높음 / HRV 낮음 → 잘 버티기 핵심 처방 (병목 플래그 ON)
- 양호 → 잘 버티기 목표치 낮게 (병목 플래그 OFF)
```

**안전장치**: 일일 칼로리 적자 1,000kcal 초과 (주 감량 ~1kg 초과) 시
- `plan.isUnsafe = true`
- "⬆ 목표 올리기" 버튼 `disabled`
- `.warning-strip.show` 경고 표시

## 넛징 시스템

### 현재 구현 상태
- `onNudgeDone(cat = 'burn')` — burn 카테고리 고정 하드코딩
- 넛징 카드 1종만 표시됨

### 개선 필요: 넛징 카드 다양화 + 맥락 바꾸기 버튼

**목적**: 데모 시 다양한 위협 시나리오를 인위적으로 전환해서 보여줄 수 있게.

**구현 방향**:
- 넛징 카드 하단 또는 우측에 "맥락 바꾸기" 버튼 추가
- 클릭 시 4개 카테고리 위협 시나리오를 순환 (burn → sleep → progress → resilience → burn...)
- 또는 pill 탭으로 카테고리 직접 선택

**카테고리별 넛징 시나리오 (각 1~2개)**:

```
🔥 잘 태우기 위협
- "최근 3일 걸음수가 목표의 60%예요. 오늘 퇴근길 한 정거장 먼저 내려보세요."
- "이번 주 운동 세션이 없어요. 10분 걷기부터 시작해보세요."

😴 잘 자기 위협
- "최근 이틀 수면이 목표보다 1.5시간 부족해요. 오늘 밤 취침 1시간 전 폰을 내려놓아 보세요."
- "수면 점수가 연속 3일 60 이하예요. 취침 전 5분 호흡 운동을 해보세요."

📉 잘 빠지기 위협
- "이번 주 감량 페이스가 목표의 절반이에요. 중간점검 사진을 올려볼까요?"
- "3일째 체중 변화가 없어요. 수분 섭취와 수면을 점검해보세요."

🧘 잘 버티기 위협
- "스트레스 지수가 이틀째 목표 상한을 넘었어요. 지금 3분 호흡 운동을 해보세요."
- "명상 세션이 일주일째 없어요. 오늘 잠들기 전 5분 바디스캔을 해보세요."
```

**`onNudgeDone(cat)` 수정**: 현재 `'burn'` 고정 → 활성 넛징 카테고리 변수로 교체

### 넛징 트리거 (플랜 위협 징후 기반)
고정 임계값이 아닌 개인 목표치 대비 이탈 감지:
- 잘 태우기: 3일 연속 소모 칼로리 목표 대비 70% 미달
- 잘 자기: 수면 점수/시간이 개인 목표치 이하 2일 연속
- 잘 빠지기: 주간 감량 페이스 목표의 50% 미달
- 잘 버티기: 스트레스 지수가 개인 목표 상한 초과 2일 연속

## 4개 카테고리

| 카테고리 | 핵심 질문 | 원시 데이터 | 목표치 결정 방식 |
|---------|---------|-----------|--------------|
| 🔥 잘 태우기 | 오늘 충분히 움직였나? | 걸음수, 운동 세션, 중강도 활동시간, 소모 칼로리, 좌식 알림 | 필요 칼로리 적자 역산 + 현재 활동량 반영 |
| 😴 잘 자기 | 회복이 됐나? | 수면 시간/단계/점수, 혈중 산소, 피부온도, 안정시 심박, HRV | 현재 수면 상태 진단 후 병목 여부에 따라 목표치 차등 |
| 📉 잘 빠지기 | 실제로 변하고 있나? | 체중(체중계 연동), 체성분, 에너지 점수 + 중간점검 사진 AI 분석 | 목표 감량 ÷ 기간으로 주간 페이스 역산 |
| 🧘 잘 버티기 | 무너지지 않고 있나? | 스트레스 지수, HRV 트렌드, 명상 세션, 호흡 운동 | 현재 스트레스/HRV 진단 후 병목 여부에 따라 목표치 차등 |

### 잘 빠지기 — 2개 서브트랙
- **수치 트랙**: 체중계 연동 → 매일 체중 기록 → 주간 이동평균 → 플랜 예측 곡선과 비교
- **시각 트랙**: 중간점검 사진 → 목표 체형과 비교 → 불일치 시 플랜 재조정 트리거

**불일치 시나리오**:
- 수치 OK + 사진 더딤 → 근손실 의심 → 운동 강도 강화 제안
- 수치 더딤 + 사진 OK → 체지방→근육 전환 중 → 플랜 유지 격려
- 둘 다 더딤 → 플랜 전면 재검토 제안

## 보상 시스템

PoC A 보상 로직 재활용. 4개 카테고리 ↔ PoC A 카테고리 1:1 매핑.

## 주요 버그 및 주의사항

- `atob()` 사용 금지 (한글 깨짐) → UTF-8 TextDecoder 기반 `b64decode()` 사용
- Blob URL 사용 금지 (로컬 파일 환경 차단) → `srcdoc` 직접 주입
- Canvas Warp: 사진 비율 유지, 최대 높이 400px 내에서 렌더링
- API 키 미설정 시 더미 플랜 자동 표시 (폴백 모드)
- localStorage 키 구버전(`poc_b_plan`, `poc_b_rewards`) 초기화 코드 존재 — 세션 시작 시 자동 클리어

## 대시보드 & 솔루션 플로우

### 탭 구조
하단 탭 2개 고정:
- **대시보드 탭** — 보상 현재 상태 (뮤지엄/지구/보드게임판)
- **솔루션 탭** — 솔루션 카드 목록 + 넛징 카드 + 나만의 솔루션 만들기

### 화면 이동 플로우
```
대시보드 탭                    솔루션 탭
[보상 현재 상태]  ←─탭전환─→  [솔루션 카드들]
                               [넛징 카드]
                               [미션 실행 ✅]
      ↓ 미션 체크 후 자동 이동
[새 보상 로드된 상태]
(신규 아이템 하이라이트)
```

미션 "했어요" 체크 → 보상 지급 → 대시보드 탭으로 자동 전환 → 해당 존에 새 아이템 하이라이트.

## 대시보드 존 구조

**솔루션 1개 = 대시보드 존 1개**
잘 태우기/잘 자기 등 카테고리는 솔루션 내부 미션 분류이며, 존 구분 기준이 아님.

```
대시보드 4개 존
├── 존 1 — 종료된 솔루션 (과거 완료) → 아이템 많이 채워진 상태로 데모 시작
├── 존 2 — 기존 솔루션 A (진행 중)   → 아이템 일부 채워진 상태
├── 존 3 — 기존 솔루션 B (진행 중)   → 아이템 일부 채워진 상태
└── 존 4 — 체중감량 솔루션 (신규)    → 비어있다가 미션 수행 후 하나씩 추가
```

데모 시 존 4가 비어있는 상태에서 미션 수행 → 보상 추가되는 훅킹 효과 연출.

### 종료된 솔루션 존 표시
- 아이템은 대시보드에 그대로 유지 (달성 기록 보존)
- 존 상단에 "완료" 배지 또는 잠금 해제 느낌의 별도 표시
- 추가 미션 없음 → 새 아이템 추가 없음

## 보상 지급 로직

**솔루션·카테고리 구분 없이 전체 풀에서 미획득 아이템 랜덤 지급.**

```
미션 "했어요" 체크
    → 전체 보상 풀에서 아직 획득하지 않은 아이템 추출
    → 랜덤 1개 선택
    → 대시보드 해당 솔루션 존에 추가
    → 보상 화면으로 자동 전환 + 신규 아이템 하이라이트
```

- 어떤 솔루션, 어떤 카테고리 미션을 수행해도 갤러리 전체가 조금씩 채워짐
- 다음에 뭐가 올지 모르는 랜덤 요소 → 훅킹 효과 강화
- 전체 풀 소진 시 처음부터 순환

### Slow Museum 통합 작업 지시

#### 1. 컨트롤 패널 제거
`Slow Museum/index.html`에서 우측 컨트롤 패널 전체 제거:
- 슬라이더 4개 (카테고리 점수 조작)
- 트리거 버튼들
- 컨트롤 패널 HTML/CSS 전체
- PoC A용 점수 연동 로직 (`onScoresApplied()` 등)

#### 2. 타이틀 변경
- `Slow Museum` → `My Care Lab`

#### 3. 하단 4개 점수 카드 교체
기존 카테고리 점수 카드 내용을 아래로 교체:

```
┌─────────────┬─────────────┐
│  1x1: New   │  1x2: 솔루션A│
│  [+] 버튼   │  타이틀 +   │
│  → 솔루션   │  이행 점수  │
│  생성 홈    │             │
├─────────────┼─────────────┤
│  2x1: 솔루션B│  2x2: 종료된│
│  타이틀 +   │  솔루션     │
│  이행 점수  │  수행 점수  │
│             │  (그레이아웃)│
└─────────────┴─────────────┘
```

- **1x1 (New)**: `+` 버튼 탭 시 → `index.html`의 솔루션 생성 홈 화면으로 전환
- **1x2**: 기존 생성된 솔루션 A — 타이틀 + 이행 점수 (더미)
- **2x1**: 기존 생성된 솔루션 B — 타이틀 + 이행 점수 (더미)
- **2x2**: 종료된 솔루션 — 타이틀 + 수행 점수 + 그레이아웃 처리

#### 4. 대시보드 씬을 index.html에 screen으로 통합
- `Slow Museum/index.html`의 Three.js 씬 코드를 `index.html`의 `screen-dashboard`로 이식
- asset 경로: `Slow Museum/assets/sculptures/` 기준으로 상대경로 유지
- 하단 탭 2개:
  - **대시보드 탭** → `screen-dashboard` (통합된 뮤지엄 씬)
  - **솔루션 탭** → `screen-home` (기존 index.html 홈)

#### 5. 화면 해상도
- 대시보드 씬: 360px × 780px (모바일 프레임 기준)
- Three.js 캔버스 크기 동적 조정 필요

#### 6. 보상 연동
