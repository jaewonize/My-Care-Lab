## 좌측 콘텐츠 구성

### 상단 — 헤더
- 서비스명: **My Care Lab**
- 서브타이틀: "에이전트와 함께 만드는 나만의 건강 솔루션"

---

### 섹션 1 — Problem
한 줄 또는 두 줄로 압축.

> AI가 주는 솔루션은 왜 안 지켜질까?
> 내가 만든 목표니까, 내가 지킨다.

기존 PoC B vs 신규 대비 2줄 정도로 간결하게.

---

### 섹션 2 — Co-creation Loop
5단계 가로 플로우 다이어그램.

```
[01 Goal Setting] → [02 Plan Design] → [03 Approve] → [04 Nudge] → [05 Reward]
목표 체형           에이전트와          플랜 견적        위협 징후     수행 체크
슬라이더 설정       플랜 co-creation    확인 + 승인      행동 제안     보상 지급
```

각 단계 카드 스타일, 단계별 포인트 컬러 구분.

---

### 섹션 3 — Tech Challenges & Next Step

#### 현재 PoC 방식 (픽셀 기반)
- Canvas per-scanline Warp로 허리/어깨/허벅지 픽셀 수평 압축
- 눈대중 감량 파라미터 산출 (슬라이더 → 체중/체지방률 선형 추정)
- 한계: 실제 체성분과 시각적 변화 간 정확도 낮음

#### Next Step (고도화 방향)
1. **2D → 3D 체형 변환**
   - 전신 사진 1~2장으로 3D 바디 메쉬 생성
   - 포즈 추정(Pose Estimation) + 뎁스 추정으로 체형 복원

2. **메쉬 기반 슬라이더 조작**
   - 슬라이더 조작 시 3D 메쉬 실시간 변형
   - 부위별(복부/허벅지/팔) 독립 조작 가능

3. **밀도맵 기반 파라미터 산출**
   - 부위별 지방/근육 밀도맵 적용
   - 메쉬 부피 변화 → 실제 체지방량/근육량 역산
   - 정확한 감량 목표 파라미터 자동 생성

4. **개인화 플랜 정밀도 향상**
   - 체성분 역산값 → 일일 칼로리 적자 정밀 계산
   - 수면/스트레스 병목 자동 진단 고도화

## 우측 모바일 화면

### 방식: iframe 임베드
`index.html`을 iframe으로 직접 심기.
로컬 서버(`python3 -m http.server` 또는 `.claude/launch.json`) 띄운 상태로 열면
GLB 에셋, 3D 애니메이션 모두 정상 로드됨.

```html
<div class="phone-frame">
  <iframe
    src="index.html"
    width="360"
    height="780"
    frameborder="0"
    scrolling="no"
  ></iframe>
</div>
```

**주의**: 브라우저별 file:// 허용 여부 상이. 크롬 기준 현재 동작 확인. 브라우저별 테스트 후 업데이트 예정.
